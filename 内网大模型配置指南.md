# 内网 OpenAI 格式大模型配置指南

本指南帮助你配置 Claude Code Router 连接内网的 OpenAI 格式大模型。

## 📋 前提条件

1. **确认你的内网模型提供 OpenAI 兼容的 API**
   - API 端点格式：`http://host:port/v1/chat/completions`
   - 支持标准的 OpenAI 请求/响应格式

2. **获取必要信息**
   - API 地址（如：`http://192.168.1.100:8000`）
   - API Key（如果有）
   - 模型名称（如：`qwen-72b`, `llama-3-70b`）

## 🔧 配置步骤

### 第一步：测试 API 连接

在配置之前，先测试你的内网模型 API 是否正常工作：

```bash
# PowerShell
curl -X POST "http://你的内网地址：端口/v1/chat/completions" `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer 你的 APIKey" `
  -d '{
    "model": "你的模型名",
    "messages": [{"role": "user", "content": "Hello"}],
    "max_tokens": 100
  }'
```

如果返回正常的响应，说明 API 工作正常。

### 第二步：创建配置文件

1. **复制示例配置**（已创建在项目目录）：
   ```bash
   # 方式 1：使用提供的示例
   cp C:\Users\wyf\Desktop\ccr\config-local-ai.example.json `
      C:\Users\wyf\AppData\Roaming\.claude-code-router\config.json
   
   # 方式 2：手动创建
   # 在 ~/.claude-code-router/config.json 创建配置
   ```

2. **编辑配置文件**，修改以下关键参数：

```json
{
  "Providers": [
    {
      "name": "local-ai",                    // 自定义名称
      "api_base_url": "http://192.168.1.100:8000/v1/chat/completions",  // ⚠️ 修改为你的地址
      "api_key": "your-api-key",            // ⚠️ 修改为你的 API Key（如无认证可留空）
      "models": ["qwen-72b"]                // ⚠️ 修改为你的模型名称
    }
  ],
  
  "Router": {
    "default": "local-ai,qwen-72b"          // ⚠️ 确保名称和模型匹配
  }
}
```

### 第三步：启动服务

```bash
# 启动 Claude Code Router
ccr start

# 或使用 Web UI 配置
ccr ui
```

### 第四步：测试连接

```bash
# 使用 ccr code 测试
ccr code

# 或直接使用 curl 测试代理
curl http://127.0.0.1:3456/v1/chat/completions `
  -H "Content-Type: application/json" `
  -d '{
    "model": "local-ai,qwen-72b",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

## 🔍 常见问题排查

### 问题 1：连接超时

**症状**：请求内网模型时超时

**解决方案**：
```json
{
  "API_TIMEOUT_MS": 120000,  // 增加超时时间到 2 分钟
  "Providers": [{
    "name": "local-ai",
    "api_base_url": "http://192.168.1.100:8000/v1/chat/completions"
  }]
}
```

### 问题 2：认证失败

**症状**：返回 401 Unauthorized

**检查**：
1. API Key 是否正确
2. 是否需要 Bearer 前缀
3. 如果不需要认证，尝试设置 `"api_key": ""`

### 问题 3：模型名称不匹配

**症状**：返回 404 或模型不存在错误

**解决**：确保 `models` 数组中的名称与实际模型名称完全一致

### 问题 4：网络无法访问

**症状**：无法连接到内网地址

**检查**：
```bash
# 测试网络连通性
ping 192.168.1.100

# 测试端口是否开放
Test-NetConnection -ComputerName 192.168.1.100 -Port 8000
```

## 🎯 高级配置

### 配置多个内网模型

```json
{
  "Providers": [
    {
      "name": "local-qwen",
      "api_base_url": "http://192.168.1.100:8000/v1/chat/completions",
      "api_key": "key1",
      "models": ["qwen-72b"]
    },
    {
      "name": "local-llama",
      "api_base_url": "http://192.168.1.101:8000/v1/chat/completions",
      "api_key": "key2",
      "models": ["llama-3-70b"]
    }
  ],
  
  "Router": {
    "default": "local-qwen,qwen-72b",
    "background": "local-llama,llama-3-70b"
  }
}
```

### 添加降级链

```json
{
  "Router": {
    "default": "local-ai,qwen-72b",
    "fallback": [
      "local-ai,qwen-72b",           // 主模型失败后重试
      "ollama,qwen2.5-coder:latest"  // 本地 Ollama 作为备选
    ]
  }
}
```

### 配置 Transformers（如需要）

如果内网模型的响应格式有细微差异，可以使用 Transformers：

```json
{
  "Providers": [{
    "name": "local-ai",
    "api_base_url": "http://192.168.1.100:8000/v1/chat/completions",
    "transformer": {
      "use": [
        ["maxtoken", {"max_tokens": 32768}],  // 限制最大 token
        "tooluse"                              // 启用工具调用支持
      ]
    }
  }]
}
```

### 使用环境变量管理 API Key

```json
{
  "Providers": [{
    "name": "local-ai",
    "api_base_url": "http://192.168.1.100:8000/v1/chat/completions",
    "api_key": "$LOCAL_AI_API_KEY"  // 从环境变量读取
  }]
}
```

然后在 `.env` 文件中设置：
```bash
LOCAL_AI_API_KEY=your-actual-key
```

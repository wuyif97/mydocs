| 方案 | 工具 | 难度 | 适用场景 |
|------|------|------|----------|
| **方案 A** | CCR 直连 | ⭐ | 模型已兼容 OpenAI 格式 |

## 🚀 方案 A：CCR 直连（最快）

### 前提条件
- ✅ 内网模型提供 **标准 OpenAI API 格式**
- ✅ 端点格式：`http://host:port/v1/chat/completions`

### 配置步骤

1. **编辑 CCR 配置** `~/.claude-code-router/config.json`:
```json
{
  "Providers": [{
    "name": "local-ai",
    "api_base_url": "http://192.168.1.100:8000/v1/chat/completions",
    "api_key": "your-key",
    "models": ["qwen-72b"]
  }],
  "Router": {
    "default": "local-ai,qwen-72b"
  }
}
```

2. **启动**:
```bash
ccr start
ccr code
```

**参考文档**: [`内网大模型配置指南.md`](内网大模型配置指南.md)

## 🚀 方案 B：LiteLLM 快速启动（推荐）⭐

### 前提条件
- ✅ 需要 Python 环境
- ✅ 模型可以是任意格式（LiteLLM 会转换）

### 快速启动（3 步）

#### 1️⃣ 安装 LiteLLM
```powershell
pip install 'litellm[proxy]'
```

#### 2️⃣ 启动代理
```powershell
litellm --model openai/qwen-72b `
  --api_base http://192.168.1.100:8000/v1 `
  --api_key sk-local-key `
  --port 4000
```

#### 3️⃣ 配置 CCR 并启动
```json
{
  "Providers": [{
    "name": "litellm",
    "api_base_url": "http://localhost:4000/v1/chat/completions",
    "api_key": "sk-local-key",
    "models": ["openai/qwen-72b"]
  }],
  "Router": {
    "default": "litellm,openai/qwen-72b"
  }
}
```

```bash
ccr start
ccr code
```

**快捷脚本**: 运行 [`start-litellm.ps1`](start-litellm.ps1)

## 🚀 方案 C：LiteLLM 配置文件（高级）

### 适用场景
- ✅ 多个内网模型
- ✅ 需要负载均衡
- ✅ 需要降级链
- ✅ 生产环境部署

### 配置步骤

#### 1️⃣ 创建 LiteLLM 配置 `config.yaml`:
```yaml
model_list:
  - model_name: qwen-72b
    litellm_params:
      model: openai/qwen-72b
      api_base: http://192.168.1.100:8000/v1
      api_key: sk-local-key
      
  - model_name: llama-3-70b
    litellm_params:
      model: openai/llama-3-70b
      api_base: http://192.168.1.101:8000/v1
      api_key: sk-another-key

litellm_settings:
  drop_params: true
  num_retries: 3
```

#### 2️⃣ 启动 LiteLLM:
```bash
litellm --config config.yaml --port 4000
```

#### 3️⃣ 配置 CCR:
```json
{
  "Providers": [{
    "name": "litellm",
    "api_base_url": "http://localhost:4000/v1/chat/completions",
    "api_key": "sk-master-key",
    "models": ["qwen-72b", "llama-3-70b"]
  }],
  "Router": {
    "default": "litellm,qwen-72b",
    "fallback": ["litellm,llama-3-70b"]
  }
}
```

**参考文件**: [`litellm-config-example.yaml`](litellm-config-example.yaml)

## 💡 常见问题速查

| 问题 | 解决方案 |
|------|----------|
| ❌ 连接超时 | 增加 `timeout` 到 300-600 秒 |
| ❌ 401 认证失败 | 检查 `api_key` 是否正确 |
| ❌ 404 模型不存在 | 确认 `model` 名称拼写 |
| ❌ 无法访问内网 | 检查防火墙和网络连通性 |
| ❌ 格式不兼容 | 使用 LiteLLM 进行协议转换 |

## 📊 架构对比

### 方案 A：CCR 直连
```
Claude Code → CCR → 内网模型
```
**优点**: 简单、延迟低  
**缺点**: 要求模型完全兼容 OpenAI 格式

### 方案 B/C：LiteLLM代理
```
Claude Code → CCR → LiteLLM → 内网模型
              (路由)   (协议转换)
```
**优点**: 支持各种格式、有重试/降级机制  
**缺点**: 多一层代理、略微增加延迟
